Overview

This web page demonstrates a modern HTML5 structure, focusing on semantic markup, accessibility, and search engine optimization. It follows best practices for building pages that are easy to navigate for both users and search engines.

Objective

The objective is to build a basic web page using only HTML5 semantic elements, with attention to accessibility for assistive technologies and discoverability by search engines.

Semantic Structure

Why Use Semantic HTML?

Semantic HTML elements like <header>, <nav>, <main>, <section>, <article>, and <footer> make your page structure clear and meaningful, both for users and machines.

Logical Heading Hierarchy

Use heading tags in order: <h1> for the main title, followed by <h2> and <h3> for subsections and details, which supports accessibility and SEO.

Accessibility Best Practices

  • Use semantic HTML5 landmarks (header, nav, main, footer, section).
  • Provide accessible navigation with aria-label where appropriate.
  • Ensure the heading order is logical for screen readers.
  • Add lang="en" and relevant roles for clarity.

SEO Basics

  • Include meaningful meta titles and descriptions.
  • Use descriptive, human-readable URLs and anchor text.
  • Write clear, relevant content in natural language.
  • Structure content with headings for crawlability.
